TASK: Audit and update project dependencies for security and maintainability.

STEPS:
1. Check for security vulnerabilities:
   - Run: npm audit (or pnpm audit / yarn audit)
   - Review critical and high severity advisories
   - Apply fixes: npm audit fix (for safe, non-breaking updates)

2. Check for outdated packages:
   - Run: npm outdated (or pnpm outdated / yarn outdated)
   - Categorize updates by risk level:
     * Patch updates (x.x.PATCH): Safe to update, bug fixes only
     * Minor updates (x.MINOR.x): Usually safe, new features, no breaking changes
     * Major updates (MAJOR.x.x): May have breaking changes, review changelog first

3. Update packages safely:
   - Update all patch versions first
   - Update minor versions for well-maintained packages (React, Next.js, Tailwind, etc.)
   - For major version updates: read the changelog/migration guide before updating
   - After each batch of updates, run the build to verify nothing broke

4. Check for unused dependencies:
   - Look for packages in package.json not imported anywhere in the codebase
   - Remove confirmed unused packages with: npm uninstall <package>
   - Be careful with packages used in config files (postcss, tailwind plugins, etc.)

5. Check for duplicate or redundant packages:
   - Multiple packages doing the same thing (e.g., axios + node-fetch + got)
   - Packages that duplicate built-in functionality (e.g., lodash.get when optional chaining works)

CONSTRAINTS:
- Make minimal, focused changes. Don't break existing functionality.
- Never update major versions of core frameworks (React, Next.js) without explicit approval.
- Don't remove packages you're unsure about; some are used in build configs or scripts.
- Update one category at a time and verify the build between updates.
- Keep the lockfile (package-lock.json / pnpm-lock.yaml) in sync.
- Don't add new dependencies as part of this task.

VERIFICATION:
- After making changes, verify by running: {build_command}
- Re-run npm audit to confirm vulnerability count decreased.
- Verify the app starts and key features work after updates.
